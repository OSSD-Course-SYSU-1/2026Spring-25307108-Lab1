# 卡片计算器 UI 优化 — 技术描述文档

## 1. 优化概述

本次 UI 优化针对 ArkTS 卡片计算器的按键布局与视觉表现进行了系统性重构。核心目标是将原本紧凑的单层 4 行等宽按键布局，升级为层次分明、操作直觉化的 5 行分区布局，同时引入差异化按键样式以强化功能区隔，提升用户在 4×4 卡片小尺寸空间内的操作效率与视觉辨识度。

## 2. 布局重构：从 4 行到 5 行

### 2.1 原版布局（优化前）

```
┌─────────────────────────┐
│    表达式显示区           │
├─────────────┬───────────┤
│ C  │ ÷ │ × │    ⌫      │  ← Row 1: 4 列等宽 (25%)
├────┴───┴───┴───────────┤
│ 7  │ 8 │ 9 │    -      │  ← Row 2: 4 列等宽
├────┴───┴───┴───────────┤
│ 4  │ 5 │ 6 │    +      │  ← Row 3: 4 列等宽
├────┴───┴───┴───────────┤
│ 1  │ 2 │ 3 │  =  │ 0(w)│  ← Row 4: 混合 1-2-3 + 等号 + 宽零
└─────────────────────────┘
```

**原版问题分析：**

| 问题 | 描述 |
|------|------|
| 按键拥挤 | 4 列等宽导致运算符与数字无视觉区分 |
| 等号位置不突出 | 等号嵌入第 4 行与数字键混排，操作路径偏长 |
| 缺乏小数支持 | 无小数点按键，仅支持整数运算 |
| 功能层级扁平 | C/÷/×/⌫ 全挤在同一行，清除与运算操作界限模糊 |

### 2.2 新版布局（优化后）

```
┌─────────────────────────┐
│    表达式显示区           │  ← 字体 30fp / 透明度 0.38
│    [MODIFIED V2 调试]    │  ← 开发调试标签（红色小字）
├─────────────┬───────────┤
│  C  │  ÷  │  ×  │  ⌫  │  ← Row 1: 功能键行 (16%)
├────┼─────┼─────┼───────┤
│  7  │  8  │  9  │  -   │  ← Row 2: 数字行 (16%)
├────┼─────┼─────┼───────┤
│  4  │  5  │  6  │  +   │  ← Row 3: 数字行 (16%)
├────┼─────┼─────┼───────┤
│  1  │  2  │  3  │  0(w)│  ← Row 4: 数字行 (16%)
├────────────┴───────────┤
│     . (50%)    │ = (50%)│  ← Row 5: 小数点 + 等号 (16%) ★ 新增
└─────────────────────────┘
```

### 2.3 布局变更对照

| 维度 | 原版 | 优化版 | 收益 |
|------|------|--------|------|
| 按键行数 | 4 行 | 5 行 | 操作分区更清晰 |
| 每行按键数 | 4-4-4-4 | 4-4-4-4-2 | 底行大按键更易触达 |
| 等号位置 | 第 4 行 (25% 宽) | 第 5 行 (50% 宽) | 独立成行，触达面积翻倍 |
| 小数点 | 无 | 第 5 行 (50% 宽) | 新增功能按键 |
| 零号按键 | 常规宽度 | 2.5 倍宽度 | 符合传统计算器布局习惯 |
| 功能键行隶属 | C/÷/×/⌫ 混合 | C/÷/×/⌫ 集中在首行 | 功能操作视觉隔离 |

## 3. 视觉样式优化

### 3.1 按键着色体系

| 按键类别 | 背景色 | 色值 | 语义 |
|----------|--------|------|------|
| 清除 C | 半透明蓝 | `#33007DFF` | 重置操作 |
| 运算符 ÷/×/-/+ | 半透明蓝 | `#33007DFF` | 二元运算 |
| 删除 ⌫ | 半透明蓝 | `#33007DFF` | 撤销操作 |
| 数字 0-9 | 浅灰 | `#F0F0F0` | 数值输入 |
| 小数点 . | 粉色 | `Color.Pink` | 小数分隔（高亮强调） |
| 等号 = | 浅灰 | `#F0F0F0` | 求值执行 |

**设计意图：**

- 半透明蓝 (`#33007DFF`) 用于所有功能键，利用冷色调与数字区的暖/中性色调形成**色温对比**，用户视线可瞬间定位功能区域
- 粉色 (`Color.Pink`) 用于小数点——作为新增功能，需要**视觉锚点**引导用户注意。粉色在所有按键中独一无二，具有强烈的**新颖性暗示**
- 浅灰 (`#F0F0F0`) 作为数字区底色，低饱和度避免视觉疲劳，符合计算器「工具属性」——数字是主角，背景不该喧宾夺主

### 3.2 按键形状与比例

| 按键 | 宽高比 `aspectRatio` | 宽度占比 | 视觉效果 |
|------|---------------------|----------|----------|
| 数字 0 | `2.5` | 25% | 横跨单列，与传统计算器一致 |
| 小数点 . | `1` | **50%** | 底行左半区，大触控面 |
| 等号 = | `1` | **50%** | 底行右半区，大拇指自然落点 |
| 其余按键 | `1` | 25% | 标准正方形按键 |

**宽度分配逻辑：**

```
Row 1-4: 25% × 4 = 100%     ← 统一的四等分栅格
Row 5:   50% × 2 = 100%     ← 底部双键二分，便于拇指操作
```

在 4×4 卡片（360dp × 360dp 有效区域）中，底部两个 50% 宽度按键的触控区域约为 **180dp × 58dp**，远超 Fitts' Law 推荐的最小触控尺寸（48dp），有效降低误触率。

### 3.3 按键圆角与间距

所有按键统一采用 `borderRadius: 20` 的圆角矩形样式，配合 `padding: 1` 的行间距，在紧凑卡片空间内形成了自然的手指分离感知，按键之间既能清晰区分，又不浪费宝贵的卡片面积。

### 3.4 显示区优化

| 属性 | 原版 | 优化版 |
|------|------|--------|
| 字体大小 | 未明确指定 | `fontSize('30')` (30fp) |
| 文本透明度 | 不透明 | `opacity(0.38)` |
| 文本对齐 | 默认 | `TextAlign.Start` (左对齐) |
| 行数限制 | 无 | `maxLines(1)` |
| 高度占比 | 未指定 | `height('20%')` (占卡片 20%) |

显示区使用 `Stack` 容器配合 `Alignment.End` 底部对齐，表达式文字呈现为半透明的次要信息层，当用户按下等号后计算结果替换表达式并以醒目样式展示，形成**输入态 → 结果态**的视觉过渡。

## 4. 交互体验优化

### 4.1 操作分区（F-Pattern 适配）

优化后的 5 行布局遵循用户在桌面卡片场景下的**自上而下**视觉扫描路径：

```
Row 1 ── 功能操作区 (C/÷/×/⌫)   → 用户的"我需要改主意/修正"区域
Row 2 ── 数字输入区 (7-8-9-)     → 上排数字
Row 3 ── 数字输入区 (4-5-6+)     → 中排数字
Row 4 ── 数字输入区 (1-2-3-0)    → 下排数字 + 高频数字 0
Row 5 ── 执行操作区 ( .  = )     → 用户的"我输入完了"区域
```

用户的操作流线：**顶部修正 → 中部输入 → 底部执行**，与阅读方向同向，无需反复跳转视线。

### 4.2 实时表达式预览

每次按键（包括小数点）触发后即时调用 `calc()` 进行求值，并将中间结果显示在表达式的叠加层中。用户无需按等号即可看到**中间计算状态**，降低输入错误的累积风险。

### 4.3 等号结果的表达式替换

按下等号后，计算结果**直接替换表达式内容**（`this.expression = this.result`），清空结果区，用户可立即基于计算结果继续下一次运算，形成流畅的**计算→结果→继续**循环，无需手动清除。

## 5. 卡片自适应设计

卡片配置使用 `autoDesignWidth: true`，设计基准宽度 720px，支持自动宽高适配。这意味着同一张卡片在不同分辨率的桌面（如标准密度 vs 高密度屏幕）上都能保持比例一致，避免按键变形或间距失调。

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `designWidth` | 720 | 设计基准宽度（px） |
| `autoDesignWidth` | true | 自动适配实际卡片宽度 |
| `defaultDimension` | 4×4 | 默认卡片尺寸 |
| `colorMode` | auto | 跟随系统明暗主题 |

## 6. UI 组件层级树

```
CardCalc (根组件)
├── Column (主容器)
│   ├── Row [MODIFIED V2]          ← 调试标签行
│   ├── Stack                      ← 表达式显示区 (20% 高度)
│   │   └── Text (expression)      ← 表达式文本（半透明）
│   ├── Row 1                      ← 功能键行 (16% 高度)
│   │   └── ForEach(calcButton1)   ← C, ÷, ×, ⌫
│   ├── Row 2                      ← 数字行 (16% 高度)
│   │   └── ForEach(calcButton2)   ← 7, 8, 9, -
│   ├── Row 3                      ← 数字行 (16% 高度)
│   │   └── ForEach(calcButton3)   ← 4, 5, 6, +
│   ├── Row 4                      ← 数字行 (16% 高度)
│   │   └── ForEach(calcButton4)   ← 1, 2, 3, 0
│   └── Row 5                      ← 执行行 (16% 高度) ★ 新增
│       ├── Button [.]             ← 小数点 (50% 宽)
│       └── Button [=]             ← 等号 (50% 宽)
```

每行高度严格占比 16% × 5 = 80%，顶部显示区 20%，合计 100% 卡片可用高度。各行高度**均匀分配**，避免内容挤压或留白不均。

## 7. 优化前后对比总结

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 按键行数 | 4 行 | 5 行（分区明确） |
| 功能键定位 | 与数字混排 | 集中在首行（冷色调） |
| 等号位置 | 第 4 行右侧 (25%) | 独立底行右侧 (50%) |
| 小数点 | 不支持 | 底行左侧 (50%，粉色高亮) |
| 零号按键 | 宽度 1x | 宽度 2.5x（传统布局） |
| 按键着色 | 统一 | 三类差异化配色 |
| 实时预览 | 部分支持 | 全按键触发实时求值 |
| 结果展示 | 无替换机制 | 等号后自动替换表达式 |
| 显示区 | 基础文本 | 半透明分层 + 左对齐 + 单行限制 |
| 卡片适配 | 固定尺寸 | 自适应宽度 + 跟随系统主题 |

## 8. 后续优化建议

1. **触觉反馈**：为按键添加 `HapticFeedback` 震动反馈，模拟物理按键触感
2. **按压动效**：使用 `stateStyles` 为按钮添加按下态缩放/变色动画
3. **暗色主题适配**：当前 `colorMode: auto` 已启用，但硬编码色值（如 `Color.Pink`、`#F0F0F0`）不会随主题切换，建议迁移到 `$r()` 资源引用
4. **横屏适配**：当前仅支持 4×4 卡片，可扩展 `supportDimensions` 支持 2×4 / 4×2 等横竖卡片形态
5. **无障碍标注**：为每个按键添加 `.accessibilityText()` 描述，支持屏幕阅读器
