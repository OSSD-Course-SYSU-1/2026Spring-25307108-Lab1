# 卡片计算器 UI 深度重构与历史记录功能 — 技术描述文档

## 1. 版本演进概述

本次迭代在上一版小数点功能基础上，对计算器进行了全量级的 UI 重构与功能扩展。核心变更包括：(1) 视觉语言从明亮风格转向暗黑主题，采用圆形按键与毛玻璃质感；(2) 按键布局从 4 行图像按键改为 5 行纯文本网格；(3) 新增百分比、双零输入等辅助按键；(4) 实现完整的历史记录系统。这是一次从「可用」到「好用」的跨越式升级。

| 版本 | UI 风格 | 按键形态 | 按键材质 | 行数 | 运算符表达 |
|------|---------|----------|----------|------|------------|
| V1（原版） | 明亮卡片 | 方形圆角 | 纯色填充 | 4 行 | 图像图标 |
| V2（上次） | 明亮卡片 | 方形圆角 | 纯色填充 | 5 行 | 图像图标 |
| **V3（本次）** | **暗黑主题** | **正圆形** | **毛玻璃 + 渐变** | **5 行** | **纯文本** |

## 2. UI 深度重构

### 2.1 视觉语言：从明亮到暗黑

```
V2 配色方案                         V3 配色方案
┌──────────────────┐               ┌──────────────────┐
│  白色/浅灰背景     │               │  #000000 纯黑背景  │
│  蓝/灰色按键       │       →       │  半透明玻璃按键    │
│  扁平颜色块       │               │  毛玻璃质感       │
│  系统默认字体     │               │  白色文字         │
└──────────────────┘               └──────────────────┘
```

**设计意图**：暗黑主题在桌面卡片场景下具备天然优势——(1) 降低卡片视觉重量，不与壁纸争抢注意力；(2) OLED 屏幕下黑色背景完全不发光，更为省电；(3) 白色文字与黑色背景的高对比度在卡片小尺寸下比浅色方案更具辨识度。

### 2.2 按键形态：从方形圆角到正圆形

| 属性 | V2 | V3 | 视觉效果 |
|------|----|----|----------|
| `borderRadius` | `20` | `'50%'` | 方形 → 正圆 |
| `backgroundColor` | 纯色 (`#33007DFF` / `#F0F0F0`) | 半透明 (`rgba(60,60,65,0.45)`) | 不透明 → 玻璃感 |
| `backdropBlur` | 无 | `8` | 无模糊 → 毛玻璃 |
| 按键间距 | `padding: 1` 固定 | `SpaceEvenly` 均匀分布 | 拥挤 → 舒展 |
| 按键宽度 | `25%` / `50%` 定宽 | `21%` + `SpaceEvenly` | 栅格约束 → 自适应间距 |
| 按键形状 | `aspectRatio: 1` (正方形) | `aspectRatio: 1` (正圆) | — |

**毛玻璃实现原理**：

```typescript
Button() {
  Text(item.display)
    .fontSize(22)
    .fontColor('#FFFFFF')
}
.width('21%')
.aspectRatio(1)
.borderRadius('50%')
.backgroundColor('rgba(60,60,65,0.45)')   // 半透明深灰基底
.backdropBlur(8)                            // CSS 级背景模糊
```

两层视觉叠加：(1) `rgba(60,60,65,0.45)` 提供 45% 不透明度的深灰底色，确保无论壁纸明暗，按键文字始终可读；(2) `backdropBlur(8)` 对按键后方的背景内容施加 8px 高斯模糊，产生磨砂玻璃的景深感。两者协同，在保持壁纸透过性的同时保证了可读性。

### 2.3 等号按键：独立视觉身份

等号按键脱离毛玻璃体系，采用**橙金渐变色**独立标识：

```typescript
.linearGradient({
  direction: GradientDirection.Right,
  colors: [['#f29938', 0.0], ['#ffb045', 1.0]]
})
```

| 属性 | 值 | 说明 |
|------|-----|------|
| 渐变方向 | `GradientDirection.Right` | 从左到右 |
| 起始色 | `#f29938` | 深橙色 |
| 终止色 | `#ffb045` | 亮金色 |
| 字体粗细 | `FontWeight.Bold` | 加粗强调 |

等号作为最高频操作键，通过色彩的唯一性（整个界面唯一的渐变/暖色元素）形成视觉锚点，用户完成输入后拇指可瞬间定位。

### 2.4 显示区重构

```
V2 显示区                           V3 显示区
┌──────────────────────┐            ┌──────────────────────┐
│ 表达式（半透明，30fp）  │            │ 历史表达式（灰色，14fp） │  ← 新增
│                      │     →      │                      │
│                      │            │ 结果/输入（白，48fp）  │  ← 加大
│                      │            │ 右对齐，最多 2 行     │
└──────────────────────┘            └──────────────────────┘
```

| 属性 | V2 | V3 |
|------|----|----|
| 主字体大小 | `30fp` | `48fp` |
| 主字体颜色 | 黑色（默认） | `#FFFFFF`（白色） |
| 字体粗细 | 默认 | `FontWeight.Bolder` |
| 文本对齐 | `TextAlign.Start` | `TextAlign.End`（右对齐） |
| 最大行数 | `1` | `2` |
| 辅助显示 | 无 | 历史表达式 `14fp` `#999999` |

显示区高度从固定 20% 改为 `layoutWeight(1)` 弹性填充，内容靠底部对齐（`FlexAlign.End`），紧贴按键区域，视线跨度更短。计算结果以 48fp 大字展示，表达式以灰色小字悬浮于结果上方，形成清晰的信息层级。

### 2.5 历史入口按钮（H 键）

右上角新增圆形 H 按钮，作为历史记录的入口：

```typescript
Stack({ alignContent: Alignment.TopEnd }) {
  Button() {
    Text('H').fontSize(28).fontColor('#FFFFFF')
  }
  .width(72).height(72)
  .backgroundColor('rgba(60,60,65,0.45)')
  .borderRadius(36)            // 72÷2 = 36，正圆
  .margin({ top: 8, right: 16 })
  .onClick(() => { this.showHistory = true; })
}
.width('100%').height(88)
```

使用 `Stack` 容器将 H 按钮固定在右上角，独立于下方的弹性显示区。按钮尺寸 72×72dp，在 4×4 卡片（约 360dp 宽）中占比 20%，够大而不突兀。

### 2.6 按钮布局重构：从图像到文本

**V2 方案**：每个按键使用 `Image` 组件加载媒体资源（`$r('app.media.ic_cal_xxx')`），需要预置 16 张切图，扩展性差。

**V3 方案**：全部按键改用 `Text` 组件，通过 `buttonData()` 返回的 `ButtonItem[]` 二维数组驱动：

```typescript
function buttonData(): ButtonItem[][] {
  return [
    [{ display: 'AC', value: 'AC' }, { display: '%', value: '%' },
     { display: '⌫', value: 'DEL' }, { display: '÷', value: '/' }],
    [{ display: '7', value: '7' }, { display: '8', value: '8' },
     { display: '9', value: '9' }, { display: '×', value: '*' }],
    [{ display: '4', value: '4' }, { display: '5', value: '5' },
     { display: '6', value: '6' }, { display: '−', value: '-' }],
    [{ display: '1', value: '1' }, { display: '2', value: '2' },
     { display: '3', value: '3' }, { display: '+', value: '+' }],
    [{ display: '00', value: '00' }, { display: '0', value: '0' },
     { display: '.', value: '.' }, { display: '=', value: '=' }]
  ];
}
```

| 优势 | 说明 |
|------|------|
| **零依赖** | 无需 `$r('app.media.xxx')` 图像资源 |
| **运算符友好** | 直接使用 Unicode（× ÷ −），无需切图 |
| **布局统一** | 所有 5 行均为 4 列等宽，消除了 V2 中第 5 行的特殊处理 |
| **可读性** | 数据与视图分离，一眼看清全部按键定义 |

### 2.7 新增辅助按键

| 按键 | Unicode | 行为 |
|------|---------|------|
| `AC` | — | 全部清除：表达式 + 结果 + 历史表达式 |
| `%` | — | 当前操作数 ÷ 100 |
| `⌫` | U+232B | 删除最后一个字符 |
| `00` | — | 双零快捷输入（智能拼接，避免前导 0 歧义） |

**`00` 按键逻辑**：

```
输入 '00'
  ├─ 表达式为空或为 '0' → 置为 '0'（避免 "000"）
  ├─ 前一个字符为运算符 → 追加 '0'（避免 "×00"）
  └─ 其他情况 → 追加 '00'（正常补零）
```

**`%` 按键逻辑**：

```
输入 '%'
  ├─ 表达式为空 → 忽略
  ├─ 向后扫描定位当前操作数
  ├─ 将该操作数 ÷ 100
  └─ 替换回表达式，实时求值
```

## 3. 历史记录功能

### 3.1 状态模型

```typescript
@State historyList: HistoryItem[] = [];    // 最多 20 条
@State historyExpr: string = '';           // 上一次计算的表达式（显示用）
@State showHistory: boolean = false;       // 历史页面/主页面切换
@State justCalculated: boolean = false;    // 刚完成计算的状态标记

interface HistoryItem {
  expression: string;   // 原始表达式（含 * / - 内部符号）
  result: string;       // 计算结果
}
```

### 3.2 记录写入流程

```
用户按下 '='
  │
  ├─[1] 执行 calc(expression) 得到结果
  ├─[2] historyList.unshift({ expression, result })  ← 头部插入
  ├─[3] 若 historyList.length > 20 → historyList.pop()  ← 淘汰最旧记录
  ├─[4] historyExpr ← convertOperatorForDisplay(expression)  ← 保存显示用表达式
  ├─[5] expression ← ''，result ← 计算结果
  └─[6] justCalculated ← true  ← 进入「刚计算」状态
```

**容量控制**：采用 FIFO 策略，上限 20 条。`unshift` 头部插入 + `pop` 尾部淘汰，保证最新记录始终在前，免除全量排序开销。

### 3.3 历史记录页面 UI

```
┌──────────────────────────┐
│  [←]   历史记录    [🗑]   │  ← 顶栏：返回 / 标题 / 清空
├──────────────────────────┤
│                          │
│  ┌────────────────────┐  │
│  │     12 + 34        │  │  ← 表达式（灰色 14fp 右对齐）
│  │       46           │  │  ← 结果（白色 28fp Bold 右对齐）
│  └────────────────────┘  │
│                          │
│  ┌────────────────────┐  │
│  │     5.5 × 3        │  │
│  │      16.5          │  │
│  └────────────────────┘  │
│                          │
│         ...              │  ← List 组件垂直滚动
└──────────────────────────┘
```

**UI 细节**：

| 元素 | 实现 | 样式 |
|------|------|------|
| 容器 | `Column` 全屏 | `backgroundColor('#000000')` |
| 顶栏 | `Row` 固定高度 | padding 顶部 32dp |
| 返回按钮 | `←` 圆形按钮 | 40×40dp 半透明深灰 |
| 清空按钮 | `🗑` 圆形按钮 | 40×40dp 半透明深灰 |
| 列表 | `List{space:12}` | 垂直滚动，间距 12dp |
| 列表项 | `ListItem → Column` | 16dp 圆角，半透明深灰 + 毛玻璃 |
| 表达式文本 | `Text` | 14fp `#999999` 右对齐 |
| 结果文本 | `Text` | 28fp `#FFFFFF` Bold 右对齐 |
| 列表宽度 | `width('90%')` | 居中缩进 |
| 列表项 key | `JSON.stringify(item) + index` | 确保正确 diff |

### 3.4 刚计算状态（justCalculated）

新增 `justCalculated` 标志位，区别用户「刚按下等号」与「正在输入表达式」两种上下文：

```
justCalculated = true 时的输入行为：
  ├─ 数字键 (0-9, 00)  → 清空，开始新表达式
  ├─ 运算符 (+-×÷)     → 从上一次结果继续运算
  ├─ 小数点 .          → 从 '0.' 开始新小数
  ├─ 等号 =            → 对历史表达式再求值一次
  ├─ 百分比 %          → 对结果取百分数
  ├─ AC                → 全部清零，退出此状态
  └─ DEL               → 全部清零，退出此状态
```

这模仿了 iPhone 原生计算器的行为：按完等号后，输入新数字就开新算式，按运算符就接续计算。

### 3.5 运算符显示转换

内部存储使用标准 ASCII 运算符（`*` `/` `-`），显示时通过 `convertOperatorForDisplay()` 转为 Unicode 美学符号：

| 内部值 | 显示值 | Unicode |
|--------|--------|---------|
| `*` | `×` | U+00D7 |
| `/` | `÷` | U+00F7 |
| `-` | `−` | U+2212 (数学减号) |

其中减号使用 `U+2212`（MINUS SIGN）而非连字符 `U+002D`（HYPHEN-MINUS），这是专业数学排版与业余文本的细节差异——前者与加号等宽、等基线对齐。

## 4. 代码质量改进

### 4.1 按钮数据驱动化

V2 使用 4 个独立函数（`calcButton1()` ~ `calcButton4()`）硬编码按钮定义，V3 统一为 `buttonData()` 返回二维数组，通过双层 `ForEach` 渲染。修改按键只需编辑数据数组，无需碰触 UI 模板。

### 4.2 类型定义集中化

```typescript
interface ButtonItem { display: string; value: string; }
interface HistoryItem { expression: string; result: string; }
type CalcHandler = (a: string, b: string) => string;
```

所有接口/类型在组件外部声明，避免 ArkTS 编译器的类型推导歧义。

### 4.3 工具函数整理

- 新增 `isDigit(ch)` — 判断数字字符，用于 `%` 和 `00` 逻辑
- `OPERATORHANDLERS` 使用类型化签名 `CalcHandler` 替代泛型 `Function`
- `calc()` 函数导出，供全局调用

## 7. 后续优化建议

1. **历史记录持久化**：当前历史记录仅存于内存（`@State`），卡片销毁后丢失。可通过 `Preferences` 或 `RelationalStore` 实现本地持久化
2. **历史项点击复用**：点击历史记录中的某条，自动将表达式回填到当前输入区
3. **按键触觉反馈**：为 `onClick` 添加 `vibrator.startVibration()` 模拟物理按键
4. **按键按压动画**：使用 `stateStyles` 为按钮添加 `scale(0.92)` 按下缩放效果
5. **横屏适配**：扩展 `supportDimensions` 支持 2×4 等横竖形态
6. **表达式校验提示**：在表达式语法错误时给出红色抖动提示，而非静默失败
